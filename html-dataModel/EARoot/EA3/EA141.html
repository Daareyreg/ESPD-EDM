usageNote-en NOTE
Details:
Constraints are requirements in themselves, since they impose prerequisites which influence the definition, use and/or fulfilment of the requirement.
They represent hard conditions such as minimum or maximum expressions which can be used to evaluate pieces of information, the required age, income, involvement in activities, etc.

An example from the eProcurement domain is a threshold as the minimum turnover required by the buying organisation to select the candidates.

Note that CCCEV does not provide any specific guidance on when which kind of Requirement should be used. Users of this vocabulary should make decisions on this topic in their specific context.
